# Proto-Language Tracker — 프로젝트 명세서 (v0.1)

> 잘 연구된 인도유럽조어(PIE)에서 **조어 변화(sound change) feature**를 학습하고,
> 이를 조어 연구가 부진한 언어(한국어 등)에 전이하여, 독립 재구된 가상 조어들 사이의
> **형태론적 동형성(morphological isomorphism)**이 **우연을 유의하게 초과하는지** 검정한다.

---

## 0. 한 문장 요약 / 비전

조어 연구가 "부진하다"는 것은 확증 증거가 없다는 뜻이다. 이 프로젝트는 정답지가 풍부한
어족(IE, Dravidian)에서 **음운 변화 채널 `f`**를 학습하고, 그 유형론적 prior로 자료가 부족한
언어군의 조어를 역추적한 뒤, **언어 부패(음운 변화)의 동형성으로부터 정보이론적 엔트로피를
역추적**하는 것을 목표로 한다.

---

## 1. 과학적 명제 (반증 가능성)

이 프로젝트의 가치는 "동형성을 발견한다"가 아니라 **"동형성이 우연을 초과하는지를 검정한다"**에 있다.
장거리 비교언어학이 사이비로 미끄러지는 유일한 이유는 우연한 유사성(chance resemblance)을
구조적 동형성으로 착각하기 때문이며, 본 설계 전체가 이 축을 중심으로 짜인다.

- **H0 (귀무가설)**: 두 현대어군 A, B에서 규칙으로 독립 역추적한 가상 조어 proto-A, proto-B의
  음운/형태 대응은, 음소 빈도를 보존한 무작위 짝짓기(null model)와 구별되지 않는다.
- **H1 (대립가설)**: proto-A ↔ proto-B의 대응이 null 분포 대비 사전 정의된 유의수준 α에서
  유의하게 규칙적이다 = **동형성 존재**.

판정은 단일 점수가 아니라 **(유의성 p-value) + (음성 대조군에서의 무유의)** 의 동시 충족으로 한다.

## 2. 핵심 개념의 조작화 (모호 → 측정 가능)

| 개념(원 표현) | 조작적 정의 / 측정량 |
|---|---|
| 조어 변화 feature | 규칙적 음운대응 규칙집합 + 신경망 채널 `f: proto → daughter` |
| 언어 부패 | 시간에 따라 음운 변화 채널을 통과하는 과정 |
| 엔트로피 | 채널 조건부 엔트로피 `H(daughter\|proto)`, 상호정보 `I(proto;daughter)` |
| 동형성 | proto-A↔proto-B 대응표의 상호정보가 null 대비 초과하는 정도 |
| 엔트로피 역추적 | 채널을 역방향 디코딩하여 조어형 사후분포 `P(proto\|daughters)` 추정 |
| 규칙성 | held-out 딸언어형 예측 정확도 + 대응의 통계적 유의성 |

## 3. 방법 — 2단계 + 1검증 (3-arm)

```
Phase 0  방법 검증 (정답 있는 어족)
  └ Indo-European, Dravidian : 파이프라인이 정설 조어를 재현하는지로 신뢰성 확보

Phase 1  조어 변화 feature 학습  ← 1차 과정
  └ (proto형 ↔ daughter형) 정렬쌍으로 채널 f 학습
     · 모델 A: 규칙 귀납 (정렬 + 음운대응 추출, white-box)
     · 모델 B: neural seq2seq (proto↔daughter, black-box; Meloni et al. 2021류)
     · 두 모델 cross-check

Phase 2  전이 + 재구 + 동형성 검정
  proto-Koreanic 을 축으로:
     ├ vs Proto-Japonic     (Transeurasian arm)
     ├ vs Proto-Dravidian   (한–타밀/드라비다 arm)
     └ vs Basque / Sumerian (음성 대조군 / negative control)
```

학습 대상은 **특정 한국어 음운법칙이 아니라**, "어떤 음 변화가 자연스럽고 어떤 것이 부자연스러운가"라는
**유형론적 prior + 채널 모델**이다. 이것이 어족을 가로지르는 전이의 타당성 근거다.

## 4. 비교 설계 — 함정 차단 규칙 (NON-NEGOTIABLE)

아마추어 장거리 비교가 무너지는 지점을 설계로 강제 차단한다.

1. **현대어↔현대어 비교 금지 → proto↔proto만 비교.** 시간심도가 다른 두 지점 비교는 반칙.
2. **차용어 · Wanderwörter · 의성의태어 · 유아어(엄마/아빠류) 제외.** lookalike 최대 오염원.
3. **결정적 판별자 = 형태론적 동형성.** 내용어 우연 일치는 흔하나, 굴절/문법 형태소와
   **패러다임 단위의 체계적 대응**은 차용·우연으로 거의 설명 불가. 한국어·드라비다·일본어는
   모두 교착·SOV·후치사형이라 형태론 비교에 유리.
4. **순열검정 + 음성 대조군 의무화.** Basque/Sumerian을 동일 파이프라인에 통과시켜
   점수가 0 근처로 나오는지로 자기검증.
5. **시간심도 비대칭 명시.** 한국어 견실 문헌은 15c(중세국어), 古代/三韓語는 단편적;
   타밀어는 Sangam(BCE 3c), Proto-Dravidian은 4000년+ 심도. 채널/엔트로피 모델에
   **시간심도를 명시적 파라미터**로 포함.

## 5. 데이터 레이어

### 5.1 정답지 어족 (Phase 0·1)
- **Indo-European**: LIV², Pokorny, Wiktionary 재구형, IE-CoR, Tower of Babel/StarLing 동원어셋.

- **Dravidian**: DEDR(Dravidian Etymological Dictionary), Krishnamurti 2003.
  타밀/말라얄람/칸나다/텔루구/툴루/브라후이 + Proto-Dravidian 재구형.

### 5.2 검정 대상 (Phase 2)
- **Koreanic**: 중세국어(훈민정음~), 방언사전(서울·경상·함경·제주), 제주어 코퍼스, 향찰/이두.
- **Japonic**: 上代 일본어, 류큐 제어(오키나와·미야코·야에야마) 사전.
- **대조군**: Basque, Sumerian(고립어/사어).

### 5.3 표기 정규화
- 모든 형태를 **IPA + 음운자질 벡터(distinctive features)**로 변환 → 거리/정렬 계산용.
- 의미는 **Concepticon / Swadesh-기반 concept ID**로 정렬 (의미 cherry-picking 방지).

## 6. 모델 명세 — 단일 생성모델 (규칙 = 잠재구조)

설계 철학(v0.3, 정정): **규칙은 별도 검증층이 아니라 하나의 베이지안 생성모델의 잠재구조다.**
이전 v0.2의 "neural 제안 → symbolic 검증 게이트" 구조는 **over-validation**이었다 — 생성모델이
이미 규칙을 잠재변수로 쓰는데, 그 출력을 같은 규칙으로 다시 검증하는 순환·이중계산이었다.
정정: symbolic·neural·Bayesian은 분리된 3층이 아니라 **한 모델의 세 측면**이다.

| 측면 | 단일 생성모델 안에서의 정체 |
|---|---|
| symbolic | 잠재변수의 **형태** — 규칙·순서가 latent 객체 (rewrite rule `A→B/X_Y`) |
| 규칙성(신문법 규율) | **prior** `P(규칙)` = 소수·어휘전반 규칙 선호 (하드 게이트 ❌ → 강한 prior ✅) |
| neural | 그 prior를 **파라미터화**하는 부품 (자연성 점수) |
| 규칙 적용 방식 | **likelihood** `P(daughter│proto,규칙)` 계산법 (POC-3: 동시/순서) |
| Bayesian | 이 모델 위의 **추론**(사후분포·엔트로피) |

생성 스토리: `proto`(잠재) → `규칙셋+순서`(잠재, neural prior) → `daughter`(관측).
역방향 추론으로 `P(proto, 규칙 | daughters)` 와 사후 엔트로피를 얻는다.

### 6.1 잠재구조 — 규칙 (symbolic)
- **표현**: 자질 기반 rewrite rule `A → B / X_Y`. 운율 참조 가능(POC-3: Verner는 강세 필요).
- **순서·상대연대**: strata + 적용모드(`simultaneous`/`sequential`). Grimm 같은 counterfeeding은
  동시 적용(POC-3 검증). 이는 **likelihood 계산법**이지 검증기가 아님.
- 비고: 규칙은 손으로 고정하지 않고 **데이터에서 추론**(아래 6.3 탐색). 확립 음법칙은 Phase 0
  gold supervision으로만 사용.

### 6.2 Prior — 규칙성 + 자연성 (neural 파라미터화)
- **규칙성 prior(신문법 규율)**: 소수·어휘전반 규칙에 높은 사전확률(sparse). "어휘 전반 일괄 적용"을
  **하드 거부권이 아니라 prior로** 인코딩 → 추론이 규칙적 해를 자동 선호. (위음성 방지: 게이트 제거)
- **자연성 prior** `P(rule | 유형론)`: 정답 있는 다어족(IE·Dravidian·Austronesian·Uralic)으로 학습한
  *어족 횡단* 점수기. end-to-end 재구기가 아니라 "이 변화가 그럴듯한가" 점수기.
- **데이터 증강**: 조어형+규칙 샘플링으로 가짜 daughter 생성(noising) → 역전(denoising) 학습으로
  데이터 병목 해소.
- **금지**: LLM을 동원어/친족 *판정자*로 쓰지 않음(어원 환각). LLM은 IPA 전사·자료 추출 잡일 한정.

### 6.3 추론 — 역추적 & 엔트로피 (Bayesian)
- **방식**: **비미분 탐색(MCMC / 프로그램 합성)**으로 규칙셋·조어형 공간 탐색, neural은 prior 점수 공급.
  이산 규칙·순서를 자연스럽게 다루고 베이지안 추론과 직결.
- **산출**: `P(proto | daughters)` 사후분포 + MAP 재구형 + **사후 엔트로피**.
  - 재구 엔트로피 = "그 부패가 얼마나 규칙적·복원가능했는가" = 프로젝트의 *엔트로피 역추적*.
  - **엔트로피 기반 기권**: 엔트로피가 임계 초과면 "무관"이 아니라 "판단 불가"로 기권(위음성 방지).
- **동형성 입력**: MAP 단일형이 아니라 **사후 불확실성을 §6.4에 전파**.

### 6.4 검증 — 위계가 다른 두 층위 (규칙 재확인 ❌)

검증은 규칙을 다시 적용하는 것이 아니라, **추론보다 상위 층위**의 두 질문이다(redundant 아님):

**층위 2 — 모델 비교 (진짜 falsifiability, 베이지안 내부)**
- **관계모델**(공통조어+음변화 채널 공유) vs **무관모델**(A·B 독립) 의 **주변우도** 비교.
- **베이즈 팩터** `BF = P(D|관계) / P(D|무관)` — 동계 가정이 데이터를 더 잘 설명하는가.
  규칙을 재확인하는 게 아니라 모델 전체를 비교하므로 over-validation 아님.

**층위 3 — 유의성 (우연 초과, 표집분포 질문)**
- **검정 통계량 `S`** = `S_phon`(대응표 상호정보) + `S_morph`(형태론 동형성, 가중↑) + `S_dist`(Mantel).
- **순열검정**: 음소빈도 보존 무작위 의미-짝짓기 ×N(≥10⁴). `p=(#{null≥관측}+1)/(N+1)`.
- **효과크기** z, **다중비교 보정**(BH/Bonferroni), **음성 대조군**(Basque/Sumerian → 무유의 자기검증).

**검정 통계량 `S` (동형성 점수)** — 단일 수치가 아니라 세 성분의 묶음:
- `S_phon` = proto-A↔proto-B 음운대응표의 **상호정보** `I(corr_A; corr_B)` (사후분포 가중).
  대응이 규칙적일수록(소수의 대응에 질량 집중) ↑.
- `S_morph` = **형태론 동형성**(결정적 판별자): 문법 형태소·패러다임 단위 정렬 점수. 가중치 상향.
- `S_dist` = **Mantel 상관**: proto-A·proto-B의 음형 거리행렬 간 상관(공변 구조).

**유의성 산출**:
- **순열검정**으로 각 성분의 null 분포 생성 — 음소 빈도 보존 무작위 의미-짝짓기 ×N(≥10⁴).
  `p = (#{null ≥ 관측} + 1) / (N + 1)`.
- **효과크기**: 관측 `S`의 null 대비 표준화값(z) 및 원단위 크기 — p와 분리 보고.
- **다중비교 보정**: 언어쌍·의미슬롯 다중검정에 BH(FDR) 또는 Bonferroni.
- **베이즈 팩터(병행)**: 베이지안 엔진 산출 — H1/H0 증거비. p와 교차확인.
- **음성 대조군**: Basque/Sumerian을 동일 파이프라인 통과 → `S` 무유의 확인(자기검증).

## 7. 평가 기준 (스스로 속지 않기)

| 단계 | 지표 | 합격 조건(초안) |
|---|---|---|
| Phase 0 | 재구형 음소 편집거리(vs 정설 조어) | IE/Dravidian에서 baseline 대비 유의 개선 |
| Phase 1 | held-out daughter 예측 정확도, 채널 엔트로피 | 채널이 우연보다 정보적 (`I>0` 유의) |
| Phase 2 | 동형성 점수 `S` + 유의성 | 아래 **다중 게이트** 동시 충족 |

### Phase 2 판정 — 다중 게이트 (단일 p-value 금지)

H1(동형성 존재) 채택은 다음을 **모두** 만족할 때만:

1. **유의성**(층위 3): 보정 후 `p < α` (성분별 순열검정, BH/Bonferroni).
2. **효과크기**: 관측 `S`의 표준화값 `z ≥ z_min` — "유의하지만 미미함" 배제.
3. **모델 비교**(층위 2): 베이즈 팩터 `BF` ≥ 임계 — 관계모델이 무관모델보다 우월.
4. **대조군 무유의**: Basque/Sumerian 등에서 `S` 무유의.
5. **기권 아님**: 사후 엔트로피 < 기권 임계 (판단불가 상태가 아님).

> 어느 하나라도 실패 → H1 보류. 대조군 유의 시 p가 작아도 **파이프라인 결함**으로 회귀.

### 위음성 통제 (False-Negative Control) — 위양성 통제와 대칭

위음성은 "무관"으로 보여 **눈에 안 띄므로** 위양성보다 위험. 순열검정은 위양성만 막고
위음성은 못 막는다(동계쌍이 더 깎여 검정력 손실). 별도 통제 필수:

- **검정력 분석(Phase 0)**: 정답 어족(IE·Dravidian)에서 모델의 gold 대응 재현율을 **시간심도별**로
  측정 → 위음성 바닥(floor)과 **검정력=0이 되는 탐지 지평(horizon)** 산출.
- **세 음성 결과 분리**: (a)진짜 무관 / (b)신호가 지평 너머 소멸 / (c)모델이 약함(버그).
  (a)(b)는 과학, (c)는 버그 — Phase 0 검정력이 셋을 가름.
- **엔트로피 기권**: 탐지력 없는 영역은 "무관" 대신 "판단 불가"로 보고.

α·z_min·BF임계·보정방식은 §11 미해결 — arm별 **사전등록(pre-registration)**으로 사전 고정
(한–타밀처럼 결과 기대 있는 arm의 확증편향 차단, R4).
